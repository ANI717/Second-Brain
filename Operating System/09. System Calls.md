
##### Modes of OS
- User Mode
- Kernel Mode

##### User Mode
- Does not have direct access to Hardware and Resources
- Crashes does not affect entire system

##### Kernel Mode
- Privileged Mode
- Have direct access to Hardware and Resources
- Crashes cause entire system collapse

##### System Call
- OS Interface for calling OS services
- User mode makes System Call when it needs to use hardware or resources
- Program switches mode from User to Kernel to execute system operations
- Once operations are complete, mode switches from Kernel to User
- Generally available as routines written in C, C++

##### Context Switching
- Switching between modes
